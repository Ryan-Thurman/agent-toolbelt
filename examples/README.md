## Code-review references

Finished reference material for shaping review quality, tone, and coverage.

- **pr-review-reference.md** — found sample command (gh-first, structured output). The *how-to-implement*.
- **code-review-best-practices.md** — Graphite: process & "what good looks like" (small PRs, turnaround, pitfalls).
- **code-review-principles-and-standards.md** — Graphite: 8 principles + reviewer checklist + coding standards + quality-issue taxonomy.
- **code-review-comments-and-tone.md** — Graphite: comment taxonomy (blocking vs nit), good/bad examples, tone rules.
- **secure-code-review.md** — Graphite + OWASP: secure-coding guidelines, OWASP review process, and code-review security practices (3 guides). Security checklist, vulnerability list, standards mapping (OWASP Top 10 / NIST / ISO 27001), risk-prioritization. Feeds the "deep" tier.
- **ai-code-security.md** — Graphite: how AI does vuln scanning + the "augment, not replace / human oversight" framing. Directly relevant — we're building an AI reviewer.
- **defect-density.md** — Graphite: quality-metric context (defects/KLOC). Peripheral to per-PR review.
